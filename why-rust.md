
# Why Rust?

I use this repository as main rust archive.

And I use Rust for other main repositories too.

Rustaceans won't wonder about why I use Rust.

But others might be curious about the choice.

## Safety

Rust is well-known for its safety. This is the main reason of why I use Rust.

I don't want to risk the crashes or bugs of my programs.

Of course, using rust does not guarantee zero crashes or zero bugs.

But there are 'significantly' less if I program with Rust.

## Performance

There are other languages which support safety too.

But the reason of why Rust is good is, because of its performance.

Rust checks the safety at compile time, not at runtime.

Other languages check the safety at runtime, so that's the difference.

## Productivity

Rust is not only safe or fast. It's also productive.

Rust supports a lot of features to increase the productivity.

The best part is Cargo, which is a package manager of Rust.

It is a lot easier to manage the dependencies with Cargo.

Also, Rust supports a lot of good syntaxes.

It's not so complex like C, C++, or Java.

## Community

If languages should be debated with only their features, safety, or performance,
there might be a better choice. (in most cases, they are harder to learn and use)

But one important good thing of Rust is its community.

Rust has a fast-growing community, and it's much easier to get help.

Also, there's not only community.

Many big tech companies are using Rust, or considering using Rust.

And that's a big help for the language itself.

Thus, in this circumstance, I decided to use Rust, for now, and for the future.
